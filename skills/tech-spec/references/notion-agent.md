# Notion Agent — 테크 스펙 저장

테크 스펙을 Notion에 저장하기 위한 에이전트 워크플로우.

---

## 전제 조건

- Notion MCP가 연결되어 있어야 한다.
- 저장할 상위 페이지 ID가 필요하다. 없으면 사용자에게 요청한다.

---

## 저장 워크플로우

### Step 1: 페이지 생성 준비

생성된 테크 스펙 마크다운에서 다음을 추출한다:

```
타이틀: [Tech Spec] {기능명} - {YYYY-MM-DD}
상태:   Draft
내용:   전체 마크다운 본문
```

### Step 2: Notion 페이지 생성

`notion-create-pages` 툴을 사용한다.

**파라미터 구성:**
```
parent_id: {사용자가 지정한 상위 페이지 ID}
title: [Tech Spec] {기능명} - {날짜}
content: {마크다운 전체}
```

**마크다운 → Notion 블록 변환 규칙:**

| 마크다운 | Notion 블록 타입 |
|----------|-----------------|
| `# 제목` | heading_1 |
| `## 제목` | heading_2 |
| `### 제목` | heading_3 |
| 일반 텍스트 | paragraph |
| `- 항목` | bulleted_list_item |
| `` ```code``` `` | code |
| `> 인용` | quote |
| `\| 표 \|` | table |

### Step 3: 생성 확인

페이지 생성 후 URL을 사용자에게 반환한다.

---

## 상위 페이지 ID 설정

사용자가 별도로 지정하지 않으면, 아래 순서로 확인한다:

1. 현재 대화에서 언급된 페이지 ID
2. 사용자에게 직접 질문: "Notion의 어느 페이지 하위에 저장할까요? 페이지 URL이나 ID를 알려주세요."

---

## 에러 처리

| 상황 | 대응 |
|------|------|
| Notion MCP 미연결 | "Notion이 연결되지 않았습니다. claude.ai 설정에서 Notion을 연결해주세요." |
| 상위 페이지 ID 없음 | 사용자에게 요청 |
| 생성 실패 | 에러 메시지 표시 후 마크다운 파일로 대체 제공 |
